# tdd — 테스트 우선 구현

`breakdown`이 만든 작업 하나를 **테스트부터** 구현한다.

## 사이클 (작업마다 반복)
1. **Red** — 인수 조건을 표현하는 **실패하는 테스트**를 먼저 쓴다.
2. **Green** — 그 테스트를 통과시키는 **최소 코드**만 쓴다.
3. **Refactor** — 테스트가 통과한 상태를 유지하며 정리한다.

## 규칙
- 테스트 없이 구현 코드를 먼저 쓰지 않는다.
- 한 번에 한 작업만. 통과 못 한 테스트를 남겨두고 다음 작업으로 가지 않는다.
- 가능하면 표준 러너로 돌린다 (예: Node는 `node --test`).

## 입력
$ARGUMENTS
