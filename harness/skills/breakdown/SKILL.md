# breakdown — 작업 분해

`spec`이 확정한 인수 조건을 **작업 단위**로 쪼갠다.

## 할 일
1. 인수 조건마다 필요한 작업을 나열한다.
2. 각 작업은 **독립적으로 테스트·검증 가능**한 크기로 만든다. 너무 크면 더 쪼갠다.
3. 작업 간 의존 순서를 표시한다.
4. 병렬 가능한 작업과 순차로만 가능한 작업을 구분한다.

## 출력
- 순서·의존성이 표시된 작업 목록
- 각 작업의 "완료 정의(done)"

각 작업은 `tdd` 단계에서 테스트 우선으로 구현된다.

## 입력
$ARGUMENTS
