# spec — 인수 조건 확정

요청받은 기능을 코드로 옮기기 전에 **검증 가능한 인수 조건**으로 바꾼다.

## 할 일
1. 요청을 한 문장 목표로 정리한다.
2. 인수 조건을 **3개 이상** 적는다. 각 조건은 "주어진 X에서 Y하면 Z" 형태로 **관찰·검증 가능**해야 한다.
3. 범위 밖(out of scope)을 명시한다.
4. 모호한 부분은 추측하지 말고 사용자에게 질문한다.

## 출력
- 목표 한 문장
- 인수 조건 목록 (각 조건이 체크 가능한 형태)
- 범위 밖 항목

이 인수 조건은 `breakdown`의 입력이자 `qa-tester`의 최종 판정 기준이 된다.

## 입력
$ARGUMENTS
